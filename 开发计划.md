# 开发计划

## 中国象棋棋盘 FEN 识别 — 开发路线图

---

### 第一阶段：跑通端到端流程（MVP）

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 1.1 | 收集/生成棋子训练数据 | 字体渲染 + 数据增强，每类 400 张，共 6000 张 | ✅ |
| 1.2 | 训练初始模型 | 运行 train.py，目标准确率 > 90% | ✅ 99.98%/100% |
| 1.3 | 收集真实棋盘照片 | 5-10 张不同角度、光照的真实照片 | ⬜ |
| 1.4 | 端到端测试 | 完整 pipeline 真实照片测试 | ⬜ |
| 1.5 | 问题修复 | 修复真实场景下的各种问题 | ⬜ |

### 第二阶段：提升准确率与鲁棒性

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 2.1 | 优化棋盘检测 | 加入棋盘线交点验证、颜色分布校验 | ⬜ |
| 2.2 | 增加棋盘类型兼容 | 木板、塑料、截图等多种样式 | ⬜ |
| 2.3 | 优化格子分割 | 棋盘线自动检测对齐，减少硬编码依赖 | ⬜ |
| 2.4 | 模型调优 | 尝试不同 backbone、超参搜索、类别平衡 | ⬜ |
| 2.5 | 数据增强管道 | 实现 generate_data.py，自动生成增强数据 | ✅ |

### 第三阶段：功能扩展

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 3.1 | 红方视角自动判断 | 自动检测棋盘方向 | ⬜ |
| 3.2 | FEN 合法性校验 | 将帅数量、棋子不重叠等校验 | ⬜ |
| 3.3 | 批处理模式 | batch.py：批量识别多张图片，输出 CSV | ⬜ |
| 3.4 | 可视化输出 | 在棋盘上叠加识别结果（棋子名 + 置信度） | ⬜ |
| 3.5 | 中间结果可调试 | --debug 模式保存各阶段中间图像 | ⬜ |

### 第四阶段：工程化与部署

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
| 4.1 | Web API 服务 | FastAPI 提供 HTTP 接口，上传图片返回 FEN | ⬜ |
| 4.2 | 模型量化与加速 | ONNX / TensorRT / OpenVINO | ⬜ |
| 4.3 | 单元测试覆盖 | pytest 为每个模块编写测试用例 | ⬜ |
| 4.4 | Docker 镜像 | Dockerfile 一键部署 | ⬜ |
| 4.5 | 移动端支持 | Core ML / TFLite 移动端集成 | ⬜ |

### 第五阶段：社区与生态

| # | 任务 | 说明 | 状态 |
|---|------|------|------|
