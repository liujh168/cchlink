# 开发计划

## 中国象棋棋盘 FEN 识别 — 开发路线图

### 当前重点：真实照片鲁棒性 + standard-v4 主线

用户已确认新的真实棋盘模板作为后续棋盘和棋局生成标准：棋子圆心落在交叉点，
红黑九宫完整 X 形斜线，楚河汉界完整，炮位与兵卒位使用紧凑直角定位标记
（距交点 6px、短臂 5px），边缘兵卒位仅绘制朝棋盘内部的双角标记。

当前主线模型为 `data/models/checkpoint_standard_v4.pth`，训练入口为
`scripts/generate_grouped_data.py` + `scripts/train_v2.py`。早期派生训练集、旧实验脚本和
历史评估 JSON 已清理；`data/pieces/` 作为基础棋子训练图块继续保留。

v4 晋级门槛：标准评测检测与网格通过率均不低于 99%，格子级准确率不低于 98.5%，
整盘完全正确率不低于 80%，三张人工确认预览全部完全正确。

`standard-v4` 当前标准评测为 5398/5400 格正确、58/60 盘完全正确，已满足标准晋级门槛。
真实照片回归集 `real-v4` 在当前推理逻辑下达到检测 18/18、网格 18/18、格子级
1441/1620（88.95%）、整盘 6/18；单模型仍不作为当前最佳主线，后续继续用
`standard_manifest.csv` 与 `real_manifest.csv` 双清单防回退。
`standard-v5` 数据生成链路已开始推进：新增真实照片风格增强、边缘有子多裁剪尺度与轻微外向偏移，
并在 manifest 中记录 `patch_scale`、`patch_shift_y/x`、`edge_augmented` 供审计和复盘；下一步是
生成 `data/pieces_grouped_v5`、训练 `checkpoint_standard_v5.pth`，再同时跑标准与真实照片清单。
单模型 `checkpoint_standard_v5.pth` 标准集提升到 5399/5400、59/60，但真实集降到
1286/1620，暂不晋级。随后加入网格交点全局中位偏移正则化，降低真实照片中棋子边缘
抢走外圈网格峰值的问题。当前较优候选是网格正则化 + v4/v5 概率融合（权重 0.9/0.1）：
标准集 5398/5400、58/60，真实集 1308/1620（80.74%）、整盘 1/18。
本轮对真实集诊断口径继续细化：315 个错误中 `piece_to_empty` 为 238 个，
其中 `edge:piece_to_empty` 为 157 个；按风格看 wood 为 171 个错误、按布局看 initial
为 226 个错误。尝试过基于边缘空判的推理端自适应多裁剪重评分，但离线回放显示能救回
少量漏判的同时会引入更多错类，因此暂不作为默认推理策略上线。下一步主线转为
`standard-v6` 数据生成：提高木质棋盘和初始局占比，增强外圈大棋子/贴边裁剪、压缩、
暗角、虚焦和不均匀光照，再训练 `checkpoint_standard_v6.pth` 并与 v4/v5 做双清单对比。
正式训练结果显示，`checkpoint_standard_v6.pth` 单模型标准集达到 5400/5400、60/60，
但真实集仍弱于融合候选，暂不单模型晋级。当前最佳候选为 v4/v5/v6/v8 概率融合
（权重 0.44/0.03/0.36/0.17）叠加近完整初始局模板补全、视觉占位补全、候选面积先验、
士/象静态点概率先验与稀疏残局身份先验：标准集 5400/5400、60/60，真实集
1546/1620（95.43%）、整盘 12/18。最佳融合将 `edge:piece_to_empty` 从 157 降到 32，
initial 真实图已全部整盘成功；`standard-v8` 单模型不晋级，但作为小剂量融合信号帮助
screen endgame 多成功 1 盘，稀疏残局身份先验进一步拉回 `real_006`。
